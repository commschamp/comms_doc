Detailed Description

template<typename TFieldBase, typename T, typename... TOptions>
class comms::field::FloatValue< TFieldBase, T, TOptions >

Field that represent floating point value.

Represents IEEE 754 floating point value, which means the value is serialised as is (consumes 4 bytes for float, and 8 bytes for double), using big or little endian.

Template Parameters
TFieldBaseBase class for this field, expected to be a variant of comms::Field.
TBasic underlying floating point type, must be either float or double.
TOptionsZero or more options that modify/refine default behaviour of the field. If no option is provided The field's value is serialised as is.
using MyFieldBase = comms::Field<comms::option::def::BigEndian>;
using MyField =comms::field::FloatValue<MyFieldBase, float>;
In the example above it will consume 4 bytes (because sizeof(float) == 4) and will be serialised using big endian notation.
Supported options are:
  • comms::option::def::DefaultValueInitialiser or comms::option::def::DefaultNumValue.
  • comms::option::def::ContentsValidator
  • comms::option::def::ValidNumValueRange, comms::option::def::ValidNumValue, comms::option::def::ValidBigUnsignedNumValueRange, comms::option::def::ValidBigUnsignedNumValue
  • comms::option::def::ValidRangesClear
  • comms::option::def::ContentsRefresher
  • comms::option::def::HasCustomRead
  • comms::option::def::HasCustomRefresh
  • comms::option::def::FailOnInvalid
  • comms::option::def::IgnoreInvalid
  • comms::option::def::Units* - all variants of value units, see Value Units for details.
  • comms::option::def::EmptySerialization
  • comms::option::def::InvalidByDefault
  • comms::option::def::VersionStorage
