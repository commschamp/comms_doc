Add "missing size" output parameter to frame's "read" operation.

Can be passed as variadic parameters to "read" functions of frame (see comms::frame::FrameLayerBase::read() and comms::frame::FrameLayerBase::readFieldsCached()). It can be used to retrieve missing length information in case "read" of frame operation returns comms::ErrorStatus::NotEnoughData.

using Frame = ...
Frame frame;
Frame::MsgPtr msg;
std::size_t missingSize = 0U;
auto es = frame.read(msg, readIter, size, comms::frame::missingSize(missingSize));
if (es == comms::ErrorStatus::NotEnoughData) {
... // missingSize will hold a minimal number of bytes that are required to be yet read
}

details::MsgIndexRetriever comms::frame::msgIndex ( std::size_t &  val)

Add "message index" output parameter to frame's "read" operation.

In case the expected input message types tuple contains multiple different message classes but having the same numeric ID, it may be beneficial to know the reletive index (starting from the first message type having the same ID) of the detected message. This function can be used to retrieve such information. Can be passed as variadic parameters to "read" functions of frame (see comms::frame::FrameLayerBase::read() and comms::frame::FrameLayerBase::readFieldsCached()).

using Frame = ...
Frame frame;
Frame::MsgPtr msg;
my_prot::MsgId msgId = my_prot::MsgId();
std::size_t msgIndex = 0U;
auto es =
frame.read(
msg,
readIter,
size,
comms::frame::msgId(msgId),
comms::frame::msgIndex(msgIndex));
if (es == comms::ErrorStatus::Success) {
assert(msg); // Message object must be created
... // use msgId and msgIndex values
}
